Help Me Decide

Two quick pickers for two common Fabric decisions. Answer the questions in each — we'll highlight the best fit.

1 Data movement Mirroring · Copy Job · Pipelines · Eventstreams

1 What's your latency need?
2 Where is your source data?
3 How much transformation do you need?
4 How much control do you want?
Our pick: Answer the questions to see a recommendation
Mirroring Simple & free

A simple and free way to mirror operational data into Fabric for analytics. Minimal setup, writes to a single read-only destination in OneLake.

  • Zero-ETL
  • OneLake replica
  • Read-only
  • Live
Learn about Mirroring ↗
Copy Job Low-code, multi-mode

Makes ingestion easier with native bulk copy, incremental copy, and CDC replication — without building pipelines, while still giving you advanced options. Many sources and destinations.

  • Bulk
  • Incremental
  • CDC
  • No pipeline needed
Learn about Copy Job ↗
Copy activity in Pipelines Full orchestration

Built for users who need orchestrated, pipeline-based ingestion workflows. Heavily customizable with transformation logic, but you manage pipeline components yourself — including incremental state.

  • Orchestration
  • Transforms
  • Customizable
  • Multi-activity
Learn about Copy activity ↗
Eventstreams Real-time ingestion

Designed for real-time ingestion, transformation and processing of streaming data. Low-latency pipelines, schema management, and routing to Eventhouse, Lakehouse, Activator, and Custom Endpoints (AMQP, Kafka, HTTP).

  • Streaming
  • Low latency
  • Routing
  • Kafka / AMQP / HTTP
Learn about Eventstreams ↗

2 Data gateway On-prem · VNet · Cloud direct

1 Where does the data live?
2 Is the source reachable from the public internet?
3 Will you run / patch a gateway server?
4 What's the source platform?
Our pick: Answer the questions to see a recommendation
On-premises data gateway Bridge to your network

A bridge between Fabric / Power BI cloud services and data behind your firewall. You install it on a Windows VM in your network. Supports many sources — SQL Server, file shares, SAP, Oracle, Teradata, and more.

  • Windows VM
  • On-prem sources
  • Standard / Personal
  • You patch it
Learn about the on-premises data gateway ↗
VNet data gateway Managed, private to Azure

A fully managed Microsoft-hosted gateway that connects to Azure data sources behind a virtual network via subnet delegation. No VM to install, no patching — and traffic stays on the Microsoft backbone.

  • Managed
  • Azure VNet
  • Private endpoint
  • No VM
Learn about VNet data gateways ↗
Cloud direct connection No gateway needed

Connect directly from Fabric to a publicly reachable cloud source — Snowflake, BigQuery, Azure SQL public endpoint, Salesforce, Dataverse, and others. Fastest setup; authentication still applies, but no gateway component to deploy or manage.

  • No gateway
  • Public endpoint
  • SaaS / cloud-native
  • Zero infra
Learn about Fabric connectors ↗
